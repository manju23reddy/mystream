Customization

• User Interface: If you want to edit the User Interface of this app, you can do so by editing the xml files included in the res/layout folder in the Android Studio’s Design tab.
Please always use the latest version of Android Studio.
• Images: You can easily change App icons, logo, buttons and other images by accessing drawable folder, its path is app/src/main/res/drawable.
For the App icons, you have to replace the png images included into all the mipmaps folders.

• Terms Of Service HTML file: You can find an HTML file called tou.html into the assets folder of this Android Studio project, that’s a file you can edit as you wish, since there is APP_NAME here and there, replace it with your app’s name and read it carefully before publishing the app on the Play Store.

IMPORTANT: You should also check the Privacy Policy statements on the bottom of that file and edit the contact email addres into your own email adress: 
  href=  "mailto:support@yourdomain.com"> support@yourdomain.com
That's important for EU GDPR laws since 25 May 2018.
In case some user will ask you to delete his/her account, you must enter your Parse Dashboard, search for its objectId or username in all Classes, and delete the rows that contain such objectId in their cells (Pointers included).

Configuration

Head over Configs.java file and check out the comments in such file which will help you editing this template.


• Add Stickers: This app contains 13 default png images as stickers, in order for you to add stickers to this app you need to add new png images in the drawable folder and name them progressively.

Example: if you want to add a 14th sticker, do the following steps:
1. Create your png image, 256x256px or 512x512px (check the existing ones as a size reference)
2. Save it into the drawable folder
3. Rename is as s13 and enter Configs.java.
4. Change the number of the following variable into 14: 
    public static int STICKERS_AMOUNT =  14

You have to repeat the steps above for all new stickers you'll add in this app. If you're wondering why your new image name is s13 and the STICKERS_AMOUNT value is 14, that's because the app creates an array of stickers and Arrays are 0-indexed, so the s0 is the 1st sticker, the s1 is the 2nd sticker, and so on. You can easily understand that the 14th sticker of the example above is at 13 index position, and that's why your Image Set is named s13 ;)

Parse Database setup

Before using the app, you must create your own App on your back4app Dashboard. The link we’ve provided above points already to the back4app's Dashboard, but in case you don’t have an account on back4app.com yet, you’ll be redirected to the Sign Up page.

Once you’ll be signed in on back4app, enter your Dashboard and simply click on the Build new app button, give your app a name and a brief description (optional), click Next and you’ll be redirected to the settings page.
Click on SETTINGS in the Core Settings box, copy the App ID and Client Key strings and replace the existing ones into Configs.java

Strings to replace in the Configs file in the app's project:
    public static String PARSE_APP_ID = "pJf7oFpFep7wDQTqEM6RDarUhuZ8hmxQ7zofOIxQ";  
    public static String PARSE_CLIENT_KEY = "1BY9Ojxt0gwMzgpGF5noXWbLc1p2fJ6Do3mrrRoD";


Please note that users can instantly report offensive/inappropriate content and users, so you should daily or weekly check your Parse Dashboard for reported content and take action for them, you can check the isReported and reportedBy columns of the Comments, Streams and User classes.
For the Streams and Comments classes, the reportedBy is an array of Users objectId's who reported content, which means that those users won't see those content, but other users will still see them, unless you will delete the reported rows in your Parse Dashboard.

PLEASE NOTE that if you'll remove a row in those classes, you will have to find all the references to its objectId in the other classes too and remove those rows where its objectId is contained.

Facebook Login setup

1. Create your own Facebook App here: Facebook developer console
2. Open your Facebook App Dashboard and select your application.
3. Select Settings from the left navigation and select Basic type Privacy Policy URL and select Category.


4. Click Add Platform at the bottom of the page and select Android.
5. Locate your package name in Android Studio and copy it to your clipboard.
6. Return to the App Dashboard and paste your package name into the Google Play Package Name field.
7. Type Intro in the Class Name filed.
8. Enable Single Sign On.
9. Now run your app via Android Studio, either on the Emulator or real device, do not login, just wait for the app to print a Key Hash in your Logcat. You should get something like this (just an example): 
    I/log-: keyhash: U05jRHlAyYvJZCAuPfKI+/l6fzQ=
10. Copy that key hash string in your clipboard and paste it in the Key Hashes filed.


11. Click Save Changes at the bottom of the App Dashboard window.
12. Enter your app Server Settings on back4app, scroll down to the Facebook Login section and click SETTINGS


13. Paste your Facebook App ID in the empty filed and click the + button, then SAVE


14. Now open strings.xml in Android Studio and replace the existing ID with your own Facebook App ID:
<string name="facebook_app_id">1373896799328580</string>


15. Lastly make your app public:


16. Lastly, when your app will be ready to be uploaded to the Play Store, generate a Signed APK and install it in your device - keep it plugged in the USB port of your computer.
Then run your Signed APK, enter the Intro screen of your app and check your Logcat to get a new key hash: you must copy it and paste it next to the previuous keyhash string you've set on step 10.

You’re done with setting up Facebook Login in this app, now you can test it.

Push Notifications setup

1. Add your project in your Firebase console by giving it a name (usually the new name you gave to your app), and click CREATE PROJECT.
2. After Firebase has created your Project, click Add Firebase to your Android app.
a) Paste your new app's package name in the first field
b) Type the name of your app in the second field.
c) Ingore ther SHA-1 field and just click REGISTER APP



3. Download the the google-services.json file, after you've downloaded it, click NEXT

4. In the Add Firebase SDK section, ingore everything and click NEXT

5. In the last section, just click Skip this step. You'll be redirected to the Overview page of your new Firebase project.


6. Click on the settings icon -> Project settings:


7. Click the CLOUD MESSAGING tab and check your Server Key and Sender ID strings, you'll need to paste them in the code and Android Push Notifications section on back4app.


8. Enter the Android Push Notifications section of your app on back4app and click EDIT. Then paste your Sender ID and Server key into their relatyive fields.
Lastly, click NEXT.


10. Open the app folder inside your Android Studio project folder and drag the google-services.json file you've previously downloaded from Firebase website, overwrite the existing one.

9. Now enter AndroidManifest.xml in Android Studio, check all the comments that start with IMPORTANT: Replace... and do what they say.


11. Enter Home.java and replace "877986214479" with your own Sender ID in this line of code, into onStart():
    // IMPORTANT: REPLACE "877986214479" WITH YOUR OWN GCM SENDER ID  
    installation.put("GCMSenderId", "877986214479");


12. Upload the main.js file into the CloudCode container of your app from your back4app dashboard. Click SETTINGS to enter it, then Choose file to upload main.js into CloudCode. Lastly, click SAVE



13. Wait for about half an hour, the time to Google to process Cloud Messaging with back4app, then you will be able to send Push Notifications to all registered devices by using the app.

One Signal setup

1. Go to One Signal O and click New App/Website.
2. Add your project in your One Signal by giving it a name (usually the new name you gave to your app), and click Google Android (FCM) and click. Configure your platform.
3. Copy server key and senderid from firebase console and paste it in server key feild and paste senderid in the senderid field and clickNEXT


4. Select Native Android and click NEXT


5. Copy your APP id and click SAVE


6. Click New Push Button


7. Add Push title in Title field and add message in Message field


8. Click Review and Send


9. Thats all, thanks
